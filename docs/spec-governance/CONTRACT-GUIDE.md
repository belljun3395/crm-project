# 백엔드 기능 계약 명세 가이드

## 1. 적용 범위

- 대상: `UseCase` 패키지와 `Domain` 패키지의 공개 API/메서드
- 목적: 구현 설명이 아니라 사용자 계약을 기록

## 2. KDoc 작성 원칙

기능 설명은 다음 항목을 최소 포함합니다.

- 기능 목적(한 줄)
- 계약 입력/전제
- 성공 시 결과
- 실패 시 예외
- 상태 변경/부작용(이벤트 발행, DB 변경, 외부 호출)

예시:

```kotlin
/**
 * UC-USER-001: 사용자 등록
 *
 * 사용자 계정 생성은 동일한 email 존재 시 실패한다.
 *
 * - 입력: name, email
 * - 성공: 사용자 엔터티 생성 및 ID 반환
 * - 실패: email 이 중복되면 DuplicateByException
 * - 부작용: UserCreated 이벤트 발행
 */
```

## 3. 테스트와의 연결

- 테스트 메소드명/설명은 동일 계약 ID를 포함
- 예시: `UC-USER-001 사용자 등록은 중복 email이면 실패한다`
- 한 UseCase 당 최소 1개 이상의 성공/실패 케이스 테스트 유지

## 4. 적용 우선순위

- 우선순위 높음: 신규 API, 핵심 도메인 규칙, 상태 전이/검증 규칙
- 중간: 단순 유틸/보조 변환 로직
